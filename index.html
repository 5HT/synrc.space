N2O Ecosystem

Synrc defines open N2O ecosystem, a full-stack specification and implementation for banking, messaging and platform development. It specifies the way to scale applications, protocols, processes, services, and storage. Ready for high-speed, low-latency IoT, MQTT, TCP, UDP and WebSocket applications, N2O powers banks, messaging systems, and several startups around America, Europe and Asia.

R&D

Synrc Research Center has founded Groupoid Infinity to research the formal languages and runtimes. We use dependent types as specification langauge to define datatypes, protocols, and interfaces. Also Groupoid Infinity is doing research in HoTT and formalization of mathematics.

Architecture

Software architecture defines services running in your environment using heterogeneous protocols and languages. Two crucial services for modern applications are messgae bus and storage services.

Locally, the message bus service could be exposed using EMQ or Mosquitto brokers, and in clouds by using MQTT IoT at AWS or Azure. The second service is a distributed database which is usually installed as DHT ring in cloud (S3, DynamoDB, Cosmos DB) or locally (Redis, SQL, Cassandra, Aerospike).

We also provide expertise in: 1) AMQP, XMPP and other less known protocols as for core bus protocols; 2) distributed databases. Depending on your needs we create simple and managable architecture, probably involving other services, like auth service, business process management, etc. The result of this phase is protocol specification for distributed application.

Prototyping

We create prototypes using Erlang language, as it allows the code to be small and concise, yet undestandable and manageable by engineers. Usually this phase can take couple months. As a result you get a demo of runnable proof of concept.
